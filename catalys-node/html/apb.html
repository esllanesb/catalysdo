Appendix B. CatalysServer Startup Arguments

The following table lists the available arguments when starting the CatalysServer class.

ParameterDescription
-xmlconfig <config-file> The path to the configuration file.

Required.

-clusterNode <cluster-node-id>

Identifies which node in a High Availability cluster to start. It must match the id of one of the ClusterNode definitions in the configuration.

Optional.

-resetseq

Reset sequence numbers for all FIX sessions. This parameter can not be used with High Availability.

Optional.

-lock

Put all FIX sessions in the locked state.

A locked state means that no incoming connection attempts will be accepted and no outbound connection attempts will be made. This state is the same as blocked except it's not persisted and only valid during the life of that FIX server instance. The lock can be released via the s_unlock command.

Optional.

-skipXmlValidation Do not perform syntactic validation of the XML configuration against the DTD file.

Semantic validation of the configuration is still performed by the Node once the XML configuration has been loaded. Specifying this option allows the server to be configured with elements and attributes which have not been specified in the DTD, which may be applicable for custom components.

Optional.

-doNotBackupConfiguration

Do not backup the configuration file when configuration changes are made at runtime.

By default the configuration is saved in XML format to a timestamped file every time the runtime configuration is changed.

Optional.

-version

Print the Node version information to stdout and then exit. When specified, all other arguments are ignored.

Optional.
